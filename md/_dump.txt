1. Video Scrubbing (GSAP ScrollTrigger)
Kenapa nggak image sequence (JPG)? Lu bilang "shot jpg". Gue sangat menyarankan jangan. 5 detik @ 30fps = 150 JPG. Preload 150 gambar = browser nangis. Pakai video MP4 yang di-scrub.
JavaScript
const video1 = document.querySelector('#video-awakening');
const video2 = document.querySelector('#video-idle');
// Video 1: Scrub via scroll
gsap.to(video1, {
currentTime: video1.duration,
ease: "none",
scrollTrigger: {
trigger: ".stage-container",
start: "top top",
end: "+=1500", // 1500px scroll untuk full transisi
pin: true,
scrub: 0.5,
onUpdate: (self) => {
// Di 80% progress, trigger transisi ke Video 2
if (self.progress > 0.8 && !transitioned) {
transitioned = true;
activateArsenal();
}
}
}
});
function activateArsenal() {
// 1. User dissolve
gsap.to('.user-particle', { opacity: 0, scale: 1.2, duration: 0.8 });
// 2. Video 1 fade out, Video 2 fade in
gsap.to('#video-awakening', { opacity: 0, duration: 1 });
gsap.to('#video-idle', { opacity: 1, duration: 1, delay: 0.5 });
// 3. Robot PNG fade in (match-cut)
gsap.to('.robot-png', {
opacity: 1,
stagger: 0.08,
duration: 0.6,
delay: 0.8
});
// 4. Enable interaksi
document.querySelectorAll('.robot-png').forEach(r => {
r.classList.add('interactive');
});
}
Critical: Video 1 harus preload full sebelum scroll. Kalau nggak, scrubbing akan patah-patah (buffering).

⚠️ CRITICAL NOTES (Jujur Soal Teknis)
1. "Shot JPG" vs Video Scrubbing
Lu bilang "shot jpg". Gue sangat menyarankan:
Jangan pakai image sequence JPG untuk scrubbing. Preload-nya berat, memory bloat, mobile bakal crash.
Pakai video MP4 yang di-scrub via currentTime. Browser handle decoding-nya lebih efisien.
Exception: Kalau lu butuh transparan (video dengan alpha channel), baru pakai image sequence PNG/WebP — tapi itu untuk objek, bukan background.
2. Match-Cut Precision
Kalau Video 1 ending dan Video 2 frame 0 tidak identik, transisi akan "jump" (patah). Solusi:
Crossfade 1 detik antara Video 1 dan Video 2. Manusia nggak notice small mismatch kalau ada crossfade.
Robot PNG fade in di tengah crossfade, jadi perhatian user ke robot, bukan ke background.
3. Robot di Video 1 vs PNG
Di Video 1, robot ada (gelap→terang). Di Fase 3, robot PNG muncul di atas Video 2.
Potential issue: Orang bisa notice kalau robot di Video 1 dan PNG "beda sedikit" (pose, lighting, detail).
Solusi:
Generate robot PNG dengan pose yang sama persis dengan robot di Video 1. Gunakan reference image dari frame Video 1 sebagai image prompt di Midjourney.
Atau: Di Video 1, robot sengaja dibuat blur/siluet saja (nggak detail), jadi orang nggak expect detail. Pas Fase 3, PNG detail muncul sebagai "surprise reveal".
4. Particle.js Performance
Kalau particle user lu kompleks (10.000 particle), di mobile bakat lag.
Solusi: Deteksi mobile → reduce particle count, atau ganti jadi CSS animation sederhana.

🔄 ALTERNATIF SIMPLER (Kalau AI Video Nggak Nurut)
Kalau Runway/Kling tetap generate robot dengan warna/terlalu detail:
Post-Process di Premiere / After Effects / CapCut:
Import video hasil AI.
Tambahkan efek "Leave Color" — hanya simpan cyan/putih, sisanya jadi hitam.
Atau: Hue/Saturation — turunin saturation semua kecuali cyan range.
Atau: Curves — crush blacks (buat gelap lebih gelap), naikin cyan highlights.
Tambahkan Glow pada area cyan/putih supaya lebih "neon".
Hasil: Video tetap hanya nunjukin cahaya cyan/putih, armor jadi hitam total.
